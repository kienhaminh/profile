## Overview

The Admin Authentication API provides secure login/logout functionality for administrators. Authentication uses HTTP-only cookies for security.

## Endpoints

### Admin Login

Authenticate an admin user and establish a session.

```http
POST /api/admin/login
```

**Request Body:**
```json
{
  "username": "admin",
  "password": "your_secure_password"
}
```

**Success Response (200):**
```json
{
  "success": true,
  "user": {
    "id": "uuid",
    "username": "admin",
    "email": "admin@example.com"
  }
}
```

This endpoint sets an HTTP-only cookie (`admin-token`) that is required for subsequent admin API calls.

### Admin Logout

Clear the admin session and remove authentication cookies.

```http
POST /api/admin/logout
```

**Response (200):**
```json
{
  "success": true
}
```

## Authentication Flow

1. **Login**: POST credentials to `/api/admin/login`
2. **Use Admin APIs**: Include the HTTP-only cookie automatically in subsequent requests
3. **Logout**: POST to `/api/admin/logout` to clear the session

## Security Features

- **HTTP-only cookies**: Prevent XSS attacks
- **Secure cookies**: HTTPS-only in production
- **CSRF protection**: Additional CSRF tokens for sensitive operations
- **Session management**: Automatic token verification and expiration

## Error Responses

**400 Bad Request:**
```json
{
  "error": "Username and password are required"
}
```

**401 Unauthorized:**
```json
{
  "error": "Invalid credentials"
}
```

**500 Internal Server Error:**
```json
{
  "error": "Internal server error"
}
```

## Cookie Details

The admin authentication cookie includes:
- **Name**: `admin-token`
- **HttpOnly**: `true` (prevents JavaScript access)
- **Secure**: `true` in production
- **SameSite**: `strict`
- **MaxAge**: 7 days

## Rate Limiting

Admin authentication endpoints are protected against brute force attacks with rate limiting.
